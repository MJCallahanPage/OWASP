What is Security Misconfiguration?

  • It is the incorrect assembly of safeguards for a web application.
  • These typically occur when there are 'holes' left in the security framework of an application by system administrators or developers.

Attackers with malicious intent can find these holes via:

  • Unauthorized access to default accounts
  • Unused webpages
  • Unpatched flaws
  • Misuse of default passwords
  • Excessive debugging information disclosure

Were an attacker to exploit such flaws, the consequences could include:

  • Data could be stolen or modified.
  • Partial or full data loss
  • Compromise of full system
  • Expensive recovery

Where could this happen?

Security misconfiguration could happen at one, many or all of these levels:

  • Operating system or platform
  • Web Server
  • Application Server
  • Database Server
  • Framework
  • Custom Code

What can be done?

Despite the ease of exploitation that Security Misconfiguration possesses, there are a number of ways to prevent such exploitation from happening.

  1. Keep all software up to date
  2. Disable any unnecessary files or features
  3. Do not present stack tracers to users
