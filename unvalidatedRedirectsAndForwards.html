What are Unvalidated Redirects and Forwards?

It is a link on a trusted site that will take the user(s) to an unauthorised site to expose a vulnerability by either installing malicious software or by gaining secure information from the user.

How is this done?

Via Redirects:

  • Similar domain names: To redirect if the URL is misspelled
  • Moving pages: Merging of sites, moving to a new domain or a name change
  • Logging out-going link: A technique that was made popular by selling ads on sites which direct them to (for example: Amazon.
  • Shortening of original link: Links on sites with hundreds and thousands of pages normally do this for shorter links

Risk Factors

There are numerous reasons unvalidated redirects are used:

  • To redirect a user to a site in order to obtain their username / password of choice.
  • To direct an end user to a site that sells services.
  • To steal information from the user.
  • Installation of malicious code

As a consequence of such risks, unvalidated redirects could result in a user/customer losing trust in the site, it could also result in a large security breach which would in turn be very costly to recover from.

What can be done?

There are relatively easy solutions to resolve unvalidated redirects.

  1. ....Just don't use redirects.
  2. Do not use parameters to calculate the destination of the user.
  3. Be sure to use the server side application for any kind of navigation.
